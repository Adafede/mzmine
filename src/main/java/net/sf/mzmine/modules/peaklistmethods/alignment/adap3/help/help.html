ADAP Aligner

This alignment algorithm has been developed as part of ADAP-GC v1.0, Automatic Data Analysis Pipeline for processing GC-MS metabolomics data.

For details, see Jiang, W.; Qiu, Y.; Ni, Y.; Su, M.; Jia, W.; Du, X.: An automated data analysis pipeline for GC-TOF-MS metabonomics studies. Journal of proteome research 2010, 9 (11), 5974-81

Description

ADAP Aligner aligns features based on their mass spectra and retention time similarity. This approach is different from Join Aligner that aligns peaks across all samples, using their m/z and retention time similarity. Instead, ADAP Aligner uses mass spectra and retention time to detect similar compounds in each sample and align them together. Due to the usage of mass spectra, this algorithm's approach to the alignment is significantly different from the approach of Join Aligner. Therefore, ADAP Aligner requires mass spectra to be constructed prior to the alignment (e.g. using Spectral Deconvolution or CAMERA).

In fact, this algorithm is similar to Hierarchical Aligner (GC), but it uses a different clustering method.

Similarity between two compounds is calculated by the following score:

S(C1, C2) = w Stime + (1 - w) Sspec

where Stime is the relative retention time difference between two compounds and Sspec is the spectrum similarity between two compounds.

Parameters

  • Min confidence (number between 0 and 1) is a fraction of the total number of samples. An aligned compound must be detected at least in several samples. This parameter determines the minimum number of samples where a compound must be detected.
  • Retention time range (minutes) is the maximum allowed retention time difference between aligned compounds from different samples.
  • M/z tolerance is the maximum m/z difference, when two peaks from different mass spectra are considered equal.
  • Score threshold (number between 0 and 1) is the minimum value of the similarity function S(C1, C2) required for compounds to be aligned together.
  • Score weight (number between 0 and 1) is the weight w that is used in the similarity function S(C1, C2).
  • Retention time similarity chooses a method used for calculating the retention time similarity. The retention time difference (fast) is preferred method.
